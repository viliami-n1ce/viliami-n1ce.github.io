Inline vs Inline-Block display in CSS

In CSS, display can have values of inline,Inline-Block and block. Containers can be given the value of each of these to produce different effects.

Inline elements have left and right margins/padding, but not top and bottom margins/paddings. This leades to inline overlapping with elements above and below it. They also allows other elements to sit to their left and right. They cannot have width or height set; they keep deafult heigh and width and flow within the flow.

Code: .red boxes{ display:block }

Block elements, however, have all margins and padding and force a line break after the block element. Blocks force a line break after the block, thus resulting in elements that are on top of each other. If width is not defined, block elements gain full width of the page.

Code: .red boxes{ display:inline }

Inline-block elements allow other elements to sit to their left and right, however unlike inline elements they can have height and width set. They also have top and bottom margins and padding, and unlike block elements they do not force a line break and this allows elements to sit to their left and right.

Code: .red boxes{ display:inline-block }
